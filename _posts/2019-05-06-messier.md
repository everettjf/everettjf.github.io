Messier can be used to trace Objective-C method calls in iOS apps. On a jailbroken device you can trace any app, and on a non-jailbroken device you can also use it to trace an app you are debugging.

<!-- more -->

# Background

In most cases, using Instruments (mainly the Time Profiler) for performance analysis of iOS apps is good enough. But the Time Profiler merges method calls together, losing the temporal (timing) representation. Then one day I saw an Android colleague using [systrace](https://developer.android.com/studio/profile/systrace.html) for performance analysis. systrace generates an HTML file that shows the time cost of functions (methods) in chronological order. I thought to myself: it would be great if iOS had a tool like this. I learned that this HTML file is generated by [catapult](https://github.com/catapult-project/catapult).

One day I saw a post on the iosre forum about [hooking objc_msgSend](http://iosre.com/t/hookzz-hack-objc-msgsend/9422). It suddenly occurred to me that I could combine catapult to generate a performance analysis chart for Objective-C methods (let's call it that for now). (Although there had always been ways to hook objc_msgSend, this time I finally couldn't resist digging into this well-cooked dish.)

So after some effort, [AppleTrace](https://github.com/everettjf/AppleTrace) was born in September 2017. While using AppleTrace, I wrote four articles introducing it:

1. [AppleTrace Performance Analysis Tool](https://everettjf.github.io/2017/09/21/appletrace/)
2. [AppleTrace with MonkeyDev to Trace Any App](https://everettjf.github.io/2017/10/12/appletrace-dancewith-monkeydev/)
3. [Installing the AppleTrace Tweak with Cydia](https://everettjf.github.io/2018/07/10/appletrace-tweak-cydia-repo/)
4. [Exploring SpringBoard with AppleTrace](https://everettjf.github.io/2018/10/28/appletrace-springboard/)

While using AppleTrace, I felt it still wasn't simple and easy enough, especially:

1. You couldn't visually define start and end points. For example, start tracing before tapping a button, and stop tracing after the button tap completes.
2. Getting the trace file was too cumbersome. You had to manually copy it out of the sandbox, then run merge.py, then run trace2html (optional), and finally open the trace file.

When the work only requires tracing one app, AppleTrace pretty much met the requirements. However, as curiosity grows and you want to look at more and more apps, the tedious "manual" operations become a bit of a hassle.


# Messier Is Born

To solve AppleTrace's usability problems, I developed the `Messier` app on top of AppleTrace (official website [https://messier-app.github.io/](https://messier-app.github.io/)). With Messier you can conveniently trace any app on a jailbroken device, and customize the trace start and end points visually more easily.

Messier consists of three parts.

1. Tweak (jailbreak plugin)
2. Dylib (dynamic library messier.framework)
3. Desktop client (Messier.dmg)

# How to Use

## Step 1: Install the macOS Client

Download and install the latest macOS client `Messier.dmg` from [here](https://github.com/messier-app/messier/releases).

![-w480](/media/15571899360772.jpg){:width="960" height="664"}




## Step 2: Jailbroken Device Configuration

(1) Installation

1. Open `Cydia`.
2. Tap `Sources` -> `Edit` -> `Add`.
3. Enter `https://messier-app.github.io/cydia`, then tap `Add Source`. After reloading you will see `Messier Repo`.
4. Go into `Messier Repo` and install the `Messier` Tweak. (Note: the Messier tweak depends on `PreferenceLoader` and `AppList`, so make sure those two Tweaks are installed first.)

(2) Configuration

Open the system Settings and scroll down to find `Messier` (under Developer). Tap to enter and start configuring, as shown below.

![-w329](/media/15571906186586.jpg){:width="658" height="1158"}


When `Enabled Applications` is turned on, it looks like the image below. You can set the apps you want to trace.

![-w327](/media/15571903092716.jpg){:width="654" height="1152"}

`Trace On App Boot` configures whether to start recording methods at app launch. Enabled by default.

`Inline Hook (HookZz)` configures whether to use Inline Hook. Enabled by default. Inline Hook is implemented using HookZz.

`Main Thread Methods Only` configures whether to record only method calls on the main thread. Disabled by default.

`Disable App Applications` configures whether to disable method recording for all apps. Disabled by default. If enabled, the `Enabled Applications` configuration will have no effect.


## Step 2: Non-Jailbroken Device Configuration

Non-jailbroken usage means working in an environment with your own source code or MonkeyDev, and requires configuring the Xcode project.

Note: first of all, Messier `only supports arm64` at the moment, so you can only use an `arm64 real device`.

Download and unzip from [here](https://github.com/messier-app/messier/releases) to get `messier.framework`.

1. Drag `messier.framework` into `Xcode Targets` -> `Build Phases` -> `Link Binary With Libraries`.
2. Click `New Copy Files Phase` to add a `Copy Files` step, drag `messier.framework` into it, and set `Destination` to `Frameworks`.

Configure as shown below:

![](/media/15571908890671.jpg){:width="1115" height="613"}


In this case, if you need to do some parameter configuration, you can go to `Project Scheme` -> `Run` -> `Arguments` and set `Environment Variables`.

```
MessierEnableOnAppBoot : true | false
MessierMainThreadMethodsOnly : true | false
```

For example:
![](/media/15571915891396.jpg){:width="736" height="373"}




## Step 3: Start Tracing

Open the macOS client. It will prompt that it is waiting for a connection, as shown below:
![-w358](/media/15571913396031.jpg){:width="716" height="486"}

Use a `USB cable` to connect the iPhone and the Mac, and open the app you have configured to trace. `If you're lucky, the app won't crash, and the macOS client will show as connected`. (If you're unlucky and the app crashes, you can disable InlineHook as described in the configuration above.)

![-w410](/media/15571913124604.jpg){:width="820" height="578"}

When tracing is finished, click `Stop`, then click `Fetch`. If you're lucky, after some file transfer, Finder will locate the `trace.json` file, as shown below:

![](/media/15571917322653.jpg){:width="1322" height="964"}

![](/media/15571917908969.jpg){:width="1172" height="430"}

## Step 4: View the Results

Open `Chrome browser` (or a Chromium-based browser) and go to `chrome://tracing`.

![](/media/15571918857092.jpg){:width="282" height="63"}

Drag in the `trace.json` produced in the previous step.

A wonderful result is born.

![](/media/15571919492832.jpg){:width="1077" height="427"}


Now you can use `w a s d` to browse this file. For detailed usage, click the `question mark in the top-right corner`, as shown:

![](/media/15571920212336.jpg){:width="756" height="563"}

# 背景

一般情况下使用Instruments（主要是Time Profiler）进行iOS 应用的性能分析就足够了，但是Time Profiler 把调用方法都合并了起来，失去了时序的表现。直到有一天看到Android开发的同事使用 [systrace](https://developer.android.com/studio/profile/systrace.html) 分析性能，systrace生成一个html文件，把函数（方法）的调用耗时按照先后顺序表现出来。心里想：要是iOS也有这样的工具就好了。了解到这个html文件是 [catapult](https://github.com/catapult-project/catapult) 生成的。

一天看到iosre论坛一篇[hook objc_msgSend](http://iosre.com/t/hookzz-hack-objc-msgsend/9422)的帖子。突然想到，可以结合catapult来生成Objective C方法的性能分析图（暂且这么叫吧）。（虽然一直也有hook objc_msgSend的方法，但这次煮好的佳肴终于忍不住下手了）。

于是经过一番努力，[AppleTrace](https://github.com/everettjf/AppleTrace) 于2017年9月份诞生了。在使用AppleTrace的过程中，我曾经写过四篇文章介绍他：

1. [AppleTrace 性能分析工具](https://everettjf.github.io/2017/09/21/appletrace/)
2. [AppleTrace 搭配 MonkeyDev Trace任意App](https://everettjf.github.io/2017/10/12/appletrace-dancewith-monkeydev/)
3. [使用 Cydia 安装 AppleTrace Tweak](https://everettjf.github.io/2018/07/10/appletrace-tweak-cydia-repo/)
4. [使用AppleTrace探索SpringBoard](https://everettjf.github.io/2018/10/28/appletrace-springboard/)

在使用AppleTrace的过程中，感受到AppleTrace还不够简单易用，尤其是：

1. 不能在视觉感受上定义开始和结束点。例如点击某个按钮之前开始Trace，按钮点击完成后停止Trace。
2. 获取trace文件太麻烦，需要手动从沙盒复制出来，然后执行merge.py，再执行trace2html（可选），最后打开trace文件。

当工作只需要Trace某一个应用时，AppleTrace几乎满足了要求，然而当好奇心增强，想看越来越多的应用时，啰嗦的“手动”操作就有点麻烦了。


# Messier 诞生

为了解决AppleTrace的易用性问题，我在AppleTrace的基础上开发了`Messier`应用（官方网站 [https://messier-app.github.io/](https://messier-app.github.io/)）。使用Messier可以方便的在越狱设备上Trace任意应用，且能更方便的在视觉上自定义Trace的开始点和结束点。

Messier由三个部分组成。

1. Tweak（越狱插件）
2. Dylib（动态库 messier.framework）
3. 桌面端（Messier.dmg)

# 使用方法

## 第一步，安装macOS客户端

在[这里](https://github.com/messier-app/messier/releases) 下载安装最新的macOS客户端 `Messier.dmg`。

![-w480](/media/15571899360772.jpg){:width="960" height="664"}




## 第二步，越狱设备配置

（1）安装

1. 打开 `Cydia`.
2. 点击 `Sources` -> `Edit` -> `Add`.
3. 输入 `https://messier-app.github.io/cydia` , 点击 `Add Source`. 重新加载后会看到 `Messier Repo`.
4. 进入 `Messier Repo`, 安装Tweak `Messier`. (注意：Messier tweak 依赖 `PreferenceLoader` 和 `AppList`，所以需要先确保这两个Tweak已经安装)

（2）配置

打开系统的设置，向下滑动可以看见`Messier`（Developer下面），点击进入即可开始配置，如下图。

![-w329](/media/15571906186586.jpg){:width="658" height="1158"}


`Enabled Applications` 打开后如下图，可设置需要Trace的应用。

![-w327](/media/15571903092716.jpg){:width="654" height="1152"}

`Trace On App Boot` 配置是否在应用启动时就开始记录方法，默认启用。

`Inline Hook (HookZz)` 是否使用Inline Hook，默认启用。Inline Hook使用了HookZz来完成。

`Main Thread Methods Only` 是否只记录主现场的方法调用，默认关闭。

`Disable App Applications` 是否关闭所有应用的方法记录，默认关闭。若启用后，`Enabled Applications`的配置将失效。


## 第二步，非越狱设备配置

非越狱使用，就是在由自己源码或者MonkeyDev的环境下进行，需要对Xcode工程配置。

注意：首先要说明的是，目前Messier`只支持arm64`，因此只能使用`arm64的真机`。

在[这里](https://github.com/messier-app/messier/releases) 下载解压后得到`messier.framework`。

1. 拖拽 `messier.framework` 到 `Xcode Targets` -> `Build Phases` -> `Link Binary With Libraries`.
2. 点击 `New Copy Files Phase` 添加 `Copy Files` 步骤, 拖拽 `messier.framework` 进去 ，配置 `Destination` 为 `Frameworks`.

如下图配置：

![](/media/15571908890671.jpg){:width="1115" height="613"}


这种情况下如果需要进行一些参数配置，可以在 `Project Scheme` -> `Run` -> `Arguments`, 设置 `Environment Variables`。

```
MessierEnableOnAppBoot : true | false
MessierMainThreadMethodsOnly : true | false
```

例如：
![](/media/15571915891396.jpg){:width="736" height="373"}




## 第三步，开始Trace

打开macOS客户端，此时会提示等待连接，如下图：
![-w358](/media/15571913396031.jpg){:width="716" height="486"}

使用`USB连接线`连接iPhone和Mac电脑，打开配置好的要Trace的应用，`运气好的话，应用不会Crash，并且macOS客户端会显示已经连接`。（如果运气不好，应用Crash了，可以按照上文配置中的说明，关闭InlineHook）

![-w410](/media/15571913124604.jpg){:width="820" height="578"}

当Trace结束，点击`Stop`，然后点击`Fetch`，如果运气好的话，一番文件传输后，会使用Finder定位到`trace.json`文件，如下图：

![](/media/15571917322653.jpg){:width="1322" height="964"}

![](/media/15571917908969.jpg){:width="1172" height="430"}

## 第四步，查看结果

打开`Chrome浏览器`（或者Chromium系列），进入`chrome://tracing`。

![](/media/15571918857092.jpg){:width="282" height="63"}

把上一步产生的`trace.json`拖拽进去。

奇妙的结果诞生了。

![](/media/15571919492832.jpg){:width="1077" height="427"}


此时，可以使用`w a s d` 来浏览这个文件了。详细使用方法可以点`右上角的问好`，如图：

![](/media/15571920212336.jpg){:width="756" height="563"}
